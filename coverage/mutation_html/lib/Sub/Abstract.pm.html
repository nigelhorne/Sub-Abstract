  101: =head1 DESCRIPTION
  102: 
  103: Enforces abstract (virtual) method contracts for plain-Perl OO without
  104: requiring Moose or Moo.  A subroutine decorated with C<:Abstract> (or
  105: named in C<use Sub::Abstract qw(...)>) is replaced at C<CHECK> time with
  106: a wrapper that C<Carp::croak>s whenever it is reached.
  107: 
  108: Perl's MRO ensures the wrapper is only reached when no subclass in the
  109: call chain has provided an implementation: if C<Dog::speak> exists, the
  110: wrapper installed in C<Animal::speak> is never called.
  111: 
  112: This module is only meaningful for plain-Perl OO or packages that do not
  113: use a full object framework.  Moo and Moose handle abstract/required
  114: methods in their own object systems.
  115: 
  116: =head2 Two usage forms
  117: 
  118: =over 4
  119: 
  120: =item Attribute form (preferred)
  121: 
  122:     sub speak :Abstract { }
  123: 
  124: The C<:Abstract> attribute is registered in C<UNIVERSAL> via
  125: L<Attribute::Handlers> when C<Sub::Abstract> is loaded, so every package
  126: has access to it without further C<use> or inheritance.  A stub body
  127: (even an empty one) is required because C<Attribute::Handlers> needs a
  128: C<CODE> ref.  The stub is replaced at C<CHECK> time.
  129: 
  130: =item Declarative form
  131: 
  132:     use Sub::Abstract qw(speak eat);
  133: 
  134: Each named method is installed as an abstract-croak wrapper at C<CHECK>
  135: time (or immediately if the module is loaded past C<CHECK>).  No stub body
  136: is needed.
  137: 
  138: =back
  139: 
  140: =head2 Bypass for testing

  263: # _wrap

  265: # Entry        : $owner_pkg -- the package that declares the abstract method

  306: =head1 KNOWN LIMITATIONS
  307: 
  308: =over 4
  309: 
  310: =item Runtime-only
  311: 
  312: Checks are runtime only.  There is no compile-time scan of C<@ISA> trees
  313: to verify that all abstract methods are implemented -- that would require
  314: knowing all subclasses at compile time, which is not possible in general Perl.
  315: 
  316: =item C<can()> returns the croak-stub
  317: 
  318: Because the stash entry is replaced with a wrapper closure,
  319: C<< Animal->can('speak') >> returns the wrapper (truthy) rather than
  320: C<undef>.  A future release may add a caller-aware C<can()> override.
  321: 
  322: =item UNIVERSAL namespace pollution
  323: 
  324: The C<:Abstract> attribute is installed in C<UNIVERSAL>, which means
  325: C<UNIVERSAL::Abstract> is added to the global namespace.
  326: 
  327: =item Not for Moo/Moose
  328: 
  329: Moo and Moose handle required/abstract methods in their own object systems.
  330: This module is for plain-Perl OO only.
  331: 
  332: =back

  410:     -- AbstractCroak: fires when the wrapper is reached (no override in MRO)
  411:     +-AbstractCroak--------------------------------------+
  412:     | Xi-Registry                                        |
  413:     | invocant? : Package                                |

  415:     | name?     : SubName                                |
  416:     |----------------------------------------------------|
  417:     | (owner?, name?) in abstract                        |
